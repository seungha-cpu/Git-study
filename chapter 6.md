# 깃허브에서 다른 사람과 소통하기  

+ 깃허브는 자신이 커밋한 소스를 보여  수 있을 뿐만 아니라 다른 개발자들과 여러 의견을 나눌 수 있는 공간이다.  
 깃허브에서 자신이 해온 활동들을 요약해 내 프로필을 관리한다면 다른 사람들이 내 저장소의 소스를 더 쉽게 파악 할 수 있다.  
 또한, 전세계에서 내로라하는 오픈 소스가 깃허브에 많이 있으므로 이 소스를 참고해서 공부 할 수 있다.  
이번 장에서는 **깃허브의 프로필 관리, README파일과 깃허브에 있는 오픈 소스와 그 사용법**에 대해서 알아보겠다.

### 컨트리뷰션(contribution)  
+ 컨트리뷰션(contribution) : 자신의 깃허브에 커밋을 올리는 것뿐만 아니라, 깃허브에서 하는 모든 활동.
+ 컨트리뷰션 그래프 (contribution graph) : 깃허브 프로필 화면에서 초록색 사각형이 찍혀있는 부분.  
컨트리뷰션 그래프는 사용자의 활동 내역이 한 눈에 보이므로 이력서 같은 역활 수행.

  
## 06-2 README 파일 작성하면서 마크다운 익히기
  
  
+ README 파일 : 공개된 깃허브 저장소는 전 세계 누구나 방문할 수 있지만 그 저장소가 어떤 곳 인지 구체적으로 알기 어렵다.  
 그래서, 방문자가 저장소 내용을 편하게 살펴볼 수 있도록 만든 안내문. 프로젝트의 기능,목적을 설명.
  
    
### 마크다운 기본 문법

 #### 제목
  
- `#`을 텍스트 앞에 붙여 제목 글자 크기를 정할 수 있음.   
- `#` 개수가 늘어날수록 글자 크기가 한단계씩 작아짐

#### 가로줄 넣기

  - 내용에 가로줄을 추가하려면 `-`나 `*`를 3개 이상 추가
 
 #### 텍스트 꾸미기

  - 굵게 : 텍스트 앞뒤로 ** 또는 __로 감싸기
  - 기울임체 : 텍스트 앞뒤로 * 또는 _로 감싸기
  - 굵은 기울임체 :  텍스트 앞뒤로 *** 또는 ___로 감싸기
  - 취소선 : 텍스트 앞뒤로 ~~로 감싸기
  
#### 소스코드
  
- 한 줄
  - 백틱 사이에 텍스트 넣기 //  `return 0`같은 형태 제작
- 여러 줄
  - 소스의 앞 뒤에 백틱 3개 붙이기. 소스 시작부분에 프로그래밍 언어(ex-bash,Python) 입력
