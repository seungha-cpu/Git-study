#깃 시작하기
  
**01-1**
  
수많은 문서를 수정하고 저장하다보면 나중에 어떤 파일에서 내용을 어떻게 수정했는지 알수 없게 된다.
깃을 이용하면 특정 문서를 언제 수정했고 어떤 것을 고쳤는지 구체적으로 관리할 수 있고, 작성한 문서를 저장 할 수 있다.
또한, 팀원들끼리 협업할 때 누가 언제 어떻게 수정했는지 알 수 있기 때문에 오류가 발생했을 때 문제원인을 파악하기 쉽다.

**01-3**
  
pwd: 현재 위치 경로 제시
ls: 현재 디렉터리에 어떤 파일이나 디렉터리가 있는지
ls -l: 상세 정보 표시
ls -al: 숨긴 파일 표시
ls -rl: 파일의 순서를 거꾸로 표시
ls -tl: 최신 파일 순으로 파일 표시
clear: 화면에 표시된 출력 제거
cd ..: 상위폴더로 이동
cd ~: 홈디렉터리로 이동
cd .: 현재 디렉터리로 이동
mkdir: 디렉터리 생성
rm: 파일이나 디렉터리 삭제 (디렉터리 삭제시 -r사용)
